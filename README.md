# Latency Test (Bash)

纯 Bash 网络延迟测试工具，无需安装 Python 或其他额外依赖。通过 TCP 连接耗时（默认 443 端口）测试到各主流服务的延迟和丢包率，并以表格形式输出，按延迟从低到高排序。

## 特性

- ✅ 纯 Bash + `curl`，无需 root 权限
- ✅ 并发测试所有目标，速度快
- ✅ 自动按延迟排序，超时/失败目标排最后
- ✅ 表格化输出，包含延迟、丢包率、状态评级、解析到的 IP

## 依赖

只依赖系统自带工具，Debian / Ubuntu 等主流发行版默认都有：

- `bash`
- `curl`
- `getent`（glibc 自带，用于域名解析）
- `awk` / `sort`

无需额外安装任何软件包。

## 用法

```bash
chmod +x latency_test.sh
./latency_test.sh
```

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-c` | 每个目标测试次数 | 3 |
| `-t` | 单次连接超时秒数 | 2 |

示例：

```bash
./latency_test.sh -c 5 -t 1.5
```

## 状态评级标准

| 延迟范围 | 状态 |
|----------|------|
| < 50ms | ✓ 优秀 |
| 50–100ms | ◆ 良好 |
| 100–200ms | ▲ 较差 |
| ≥ 200ms 或丢包率 > 50% | ✗ 异常 |
| 全部失败 | ✗ 超时 |

## 自定义测试目标

脚本内 `TARGETS` 数组可自行增删，格式为：

```bash
"显示名称|域名|端口"
```

例如新增一个目标：

```bash
"Reddit|reddit.com|443"
```

## 原理说明

脚本使用 `curl` 的 `time_connect` 指标测量 TCP 三次握手耗时，作为延迟近似值（不依赖 ICMP ping，因此不需要 root 权限，也适用于禁止 ping 的网络环境）。域名解析使用 `getent ahostsv4` 获取 IPv4 地址。
